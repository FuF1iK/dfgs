# Discord HTML Watcher

Бот для Discord, который по ссылке делает живой скриншот HTML-страницы
и публикует его в виде embed-сообщения в текстовом канале, периодически
обновляя (редактируя) это сообщение новым скриншотом.

## Команды

- `/watch url interval` — начать наблюдение со скриншотом. `interval` —
  секунды между обновлениями (минимум 10, по умолчанию 60).
- `/watchtable url selector interval` — найти на странице `<table>` (или
  таблицу по указанному CSS-селектору) и показывать её как текстовую
  таблицу в embed, обновляя по таймеру.
- `/watchdata url row_selector columns headers interval` — собрать свою
  таблицу из произвольных блоков на странице (даже без тега `<table>`).
- `/unwatch message_id` — остановить любое наблюдение по ID сообщения.
- `/listwatches` — список активных наблюдений (живёт только пока бот запущен).

### Как работает `/watchdata`

Если на сайте нет тега `<table>` (так сделано на большинстве современных
сайтов — карточки товаров, списки, плитки и т.п.), `/watchtable` не
сработает. `/watchdata` решает это: вы сами объясняете боту, где искать
данные.

**Параметры:**
- `row_selector` — CSS-селектор, который находит *каждый повторяющийся
  блок* (одну "строку" будущей таблицы). Например, если на странице список
  товаров и у каждой карточки класс `product-card`, то это `.product-card`.
- `columns` — CSS-селекторы *внутри* каждого такого блока, через запятую —
  каждый превратится в свою колонку. Например `.name, .price, .status`.
  Если в самом блоке нужного текста сразу всё значение (без вложенных
  элементов), используйте `.` в качестве селектора колонки — это означает
  "взять весь текст блока".
- `headers` (необязательно) — названия колонок через запятую, например
  `Товар, Цена, Статус`. Количество должно совпадать с количеством `columns`.

**Как найти селекторы (через DevTools браузера):**
1. Откройте нужную страницу в Chrome/Edge, нажмите `F12` (или ПКМ →
   "Просмотреть код / Inspect").
2. Кликните на инструмент "выбрать элемент" (иконка курсора в углу панели
   DevTools) и кликните по одной из повторяющихся карточек на странице.
3. В подсветившемся HTML найдите общий класс или тег этой карточки — это
   и есть `row_selector` (например `.product-card`, `li.item`).
4. Затем кликните на конкретные значения внутри карточки (название, цена)
   и так же посмотрите их класс/тег — это ваши `columns`.

Пример вызова в Discord:
```
/watchdata url:https://example.com/shop row_selector:.product-card columns:.title,.price headers:Товар,Цена
```

### Как работает `/watchtable`

Discord не умеет рисовать настоящие HTML-таблицы внутри embed, поэтому бот
строит её текстом в монospace-блоке (с выравниванием столбцов через
пробелы) — выглядит как таблица, но это просто текст с отступами.

- Если на странице несколько `<table>`, по умолчанию берётся первая.
  Чтобы указать конкретную, передайте параметр `selector`, например
  `#price-table` или `.data-table:nth-of-type(2)` (открыть DevTools на
  сайте → ПКМ по таблице → Inspect → Copy selector).
- Таблица обрезается до 15 строк и ~20 символов на ячейку, чтобы влезть в
  лимит embed (4096 символов) — для больших таблиц это даёт читаемый
  фрагмент, а не «простыню».
- Если на странице таблица рисуется через JavaScript (а не статичный HTML),
  бот всё равно её увидит — он ждёт полной загрузки страницы в браузере
  перед чтением DOM.

## Сайты с вкладками/переключателями (параметр `clicks`)

Некоторые сайты показывают нужные данные только после того, как пользователь
сам кликнет переключатель периода или вкладку (например "Неделя", "Игроки").
Параметр `clicks` есть у всех трёх команд (`/watch`, `/watchtable`,
`/watchdata`) — это список текстов кнопок через запятую, которые бот сам
нажмёт по порядку перед тем, как прочитать страницу.

**Готовый пример для статистики семьи Majestic RP (fletcher-wiki.com):**
```
/watchtable url:https://fletcher-wiki.com/players-family-stats/family/1983?server=ru7 clicks:Неделя,Игроки interval:60
```
Это покажет именно таблицу игроков (#, Игрок, Ср. урон, Общий урон,
Убийства, Каптов) за выбранную неделю, и будет обновлять её каждую минуту.

Если на каком-то сайте при первой загрузке выходит экран вида "Проверяем
ваш браузер..." (анти-DDoS защита) — это нормально, бот сам подождёт
автоматический переход дальше. Куки этой проверки сохраняются между
обновлениями (бот держит один и тот же "сеанс браузера" всё время, пока
работает), поэтому повторно проходить проверку при каждом обновлении не
требуется.

## Ограничение команд по ролям

По умолчанию команды доступны всем участникам сервера. Чтобы разрешить их
только определённым ролям (например, модераторам):

1. В Discord включите режим разработчика: Настройки пользователя →
   Расширенные → "Режим разработчика" (вкл).
2. В настройках сервера → Роли, кликните правой кнопкой по нужной роли →
   "Копировать ID роли".
3. В файле `.env` впишите этот ID в `ALLOWED_ROLE_IDS`. Если ролей
   несколько — перечислите через запятую без пробелов:
   ```
   ALLOWED_ROLE_IDS=123456789012345678,987654321098765432
   ```
4. Перезапустите бота (`Ctrl+C` в окне с `npm start`, затем снова
   `npm start`).

Если оставить `ALLOWED_ROLE_IDS` пустым — ограничение не действует, команды
доступны всем, как раньше.

## Установка

1. **Создайте приложение бота**
   - Зайдите на https://discord.com/developers/applications → New Application.
   - В разделе Bot нажмите Reset Token и скопируйте токен.
   - В разделе OAuth2 скопируйте Client ID.
   - Сгенерируйте инвайт-ссылку: OAuth2 → URL Generator → scopes `bot` и
     `applications.commands`; permissions: `Send Messages`, `Embed Links`,
     `Attach Files`, `Use Slash Commands`. Откройте полученную ссылку и
     добавьте бота на свой сервер.

2. **Установите зависимости**
   ```bash
   cd discord-html-watcher
   npm install
   npx playwright install --with-deps chromium
   ```
   Флаг `--with-deps` ставит системные библиотеки, нужные headless-браузеру
   (актуально для Linux/VPS; на Windows/macOS можно без него).

3. **Настройте переменные окружения**
   ```bash
   cp .env.example .env
   ```
   Откройте `.env` и впишите `DISCORD_TOKEN` и `CLIENT_ID`.

4. **Зарегистрируйте слэш-команды** (один раз, после любых изменений
   списка команд)
   ```bash
   npm run deploy
   ```

5. **Запустите бота**
   ```bash
   npm start
   ```

## Важные ограничения

- **Состояние не сохраняется.** При перезапуске бота все активные
  наблюдения теряются (нужно заново вызывать `/watch`). Если нужна
  персистентность между перезапусками — можно дописать сохранение в
  JSON-файл или БД, скажите, и я добавлю.
- **Не все сайты дают себя скриншотить.** Сайты с защитой от ботов
  (например, Cloudflare-challenge), требующие логин, либо рисующие контент
  через сложный JS с задержкой, могут отдавать пустую/неполную картинку.
  При необходимости можно увеличить таймауты или добавить ожидание
  конкретного элемента на странице.
- **Rate limits Discord.** Слишком частое редактирование одного сообщения
  (значительно чаще, чем раз в 5–10 секунд) может привести к ограничению
  скорости запросов. Интервал по умолчанию (60 сек) безопасен.
- **Ресурсы сервера.** Headless Chromium держит более-менее ощутимый объём
  RAM (~150–300 МБ на вкладку). Если будет много параллельных `/watch`,
  учитывайте это при выборе хостинга (VPS, не бесплатный shared-хостинг).
- Бот должен работать **постоянно** (на VPS, Raspberry Pi, и т.п.), а не
  только пока ваш компьютер включён — иначе обновления прекратятся.
